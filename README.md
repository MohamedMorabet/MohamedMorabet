### Engineering Principles

- **Readability over cleverness.** Code is read far more often than it's written.
- **Build for failure.** Handle errors explicitly. Test the unhappy paths.
- **Understand the layer below.** Knowing how your abstractions work underneath makes you a better engineer.
- **Ship, then refine.** Iterate based on real feedback rather than imagined perfection.
